## 工具简介

HSL/RGB颜色命名工具是一款能够将颜色数值转换为直观中文名称的实用软件。它支持两种颜色模式：
- **RGB模式**：红(Red)、绿(Green)、蓝(Blue)三原色数值表示
- **HSL模式**：色相(Hue)、饱和度(Saturation)、亮度(Lightness)表示

无论是设计师、开发者还是普通用户，都可以通过本工具快速获取颜色的中文名称，便于沟通和记录。

## 基本概念

### 简单解释
- **RGB**：通过红、绿、蓝三种颜色的混合来表示颜色，每种颜色数值范围0-255
- **HSL**：通过颜色的"样子"(色相)、"鲜艳程度"(饱和度)和"明暗"(亮度)来表示颜色

### 专业解释
- **RGB**：加色模型，通过不同强度的红绿蓝光混合产生各种颜色
- **HSL**：圆柱坐标系中的颜色表示方法，更符合人类对颜色的感知
  - 色相(H)：0-360度，表示颜色在色轮上的位置
  - 饱和度(S)：0-100%，表示颜色的鲜艳程度
  - 亮度(L)：0-100%，表示颜色的明暗程度

## 功能使用

### 单个颜色命名

#### 简单步骤
1. 选择颜色格式(RGB或HSL)
2. 输入颜色数值
3. 点击"获取颜色名称"
4. 查看结果和颜色预览

#### 示例
- **RGB示例**：
  - 输入：R=255, G=0, B=0
  - 结果：颜色名称: 鲜艳红
- **HSL示例**：
  - 输入：H=120, S=100, L=50
  - 结果：颜色名称: 鲜艳绿

### 批量处理

#### CSV文件格式要求
- 每行代表一个颜色
- 前三列为颜色数值(RGB或HSL)
- 无表头

#### 操作步骤
1. 准备CSV文件
2. 点击"导入CSV"
3. 选择文件
4. 处理完成后点击"导出CSV"保存结果

#### 示例CSV内容(RGB)
```
255,0,0
0,255,0
0,0,255
128,128,128
```

#### 示例CSV内容(HSL)
```
0,100,50
120,100,50
240,100,50
0,0,50
```

## 专业说明

### 命名算法详解

本工具采用基于HSL空间的精细化命名算法，主要逻辑如下：

1. **明度判断**：
   - 亮度<10%：直接判定为"黑色"
   - 亮度>90%且饱和度<10%：直接判定为"白色"
   - 饱和度<10%：根据亮度返回"深灰"/"灰色"/"浅灰"

2. **饱和度描述**：
   - <30%：添加"淡"前缀
   - >80%：添加"鲜艳"前缀

3. **亮度描述**：
   - <30%：添加"暗"前缀
   - >70%：添加"亮"前缀

4. **色相分段**：
   - 15°：红
   - 45°：橙红
   - 60°：橙
   - 90°：黄橙
   - 120°：黄绿
   - 180°：绿
   - 210°：青绿
   - 240°：青
   - 270°：蓝青
   - 300°：蓝
   - 330°：紫蓝
   - 360°：紫红

### 颜色空间转换

RGB与HSL之间的转换使用Python标准库colorsys实现：
- `rgb_to_hls()`：将RGB转换为HLS
- `hls_to_rgb()`：将HLS转换为RGB

注意：HLS与HSL略有不同，L(亮度)和S(饱和度)的计算顺序有差异。

## 常见问题

### 一般问题
**Q：为什么同样的RGB值在不同设备上看起来不一样？**
A：这是由于设备色域、校准和环境光不同造成的，本工具提供的是理论值。

**Q：能识别所有颜色吗？**
A：工具能覆盖大多数常见颜色，但极端或非常特殊的颜色可能归类为"未知颜色"。

### 技术问题
**Q：导入CSV时出现错误怎么办？**
A：请检查：
1. 文件是否为纯文本CSV格式
2. 数值是否在有效范围内
3. 是否有足够多的列(至少3列)

**Q：HSL和HSV有什么区别？**
A：HSL和HSV都是圆柱坐标系颜色模型，但计算方法不同。HSL更强调亮度变化，而HSV更强调颜色本身。

### 专业问题
**Q：命名算法是否有科学依据？**
A：算法基于人类对颜色的感知研究，参考了色彩学和心理学相关理论，但颜色命名本身具有一定主观性。

**Q：如何处理色盲用户的需求？**
A：当前版本未专门优化色盲支持，但可以通过精确的数值输入来规避颜色识别问题。
